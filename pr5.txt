PR: Plop ND→D Loop-Splice Class System

Add a full nondeterminism-to-determinism classifier that decides whether Prolog choicepoint code can be converted 1:1 into deterministic loops, maps, folds, flatmaps, spliced outputs, or memoised dependent loops without changing meaning.

The classifier should favour O(1)-style indexed recognition where possible:
1. Build signature index once.
2. Lookup known patterns by signature.
3. Only fall back to deeper structural analysis when the signature is unknown.
Required safety rules:
Do not transform if the body contains:
- cut
- assert/retract
- IO
- random
- var/nonvar-sensitive logic
- call/N unless declared safe
- once/1 unless semantics are preserved
- negation-as-failure unless explicitly supported
- disjunction unless converted to an equivalent tagged branch structure
Required semantic checks:
The optimiser must verify:
1. Same output variables.
2. Same answer order where order matters.
3. Same multiplicity where duplicates matter.
4. Same failure behaviour.
5. Same determinism after conversion.
6. Same template construction.
7. No accidental loss of nondeterministic answers.
8. No repeated expensive call if safely avoidable.
9. No hoisting across unsafe dependency boundaries.
Required example 1: map-compatible nondeterminism
Input:
collect(Xs, Ys) :-
    findall(Y, (member(X, Xs), f(X, Y)), Ys).
Output:
collect(Xs, Ys) :-
    collect_loop(Xs, [], Rev),
    reverse(Rev, Ys).
collect_loop([], Acc, Acc).
collect_loop([X|Xs], Acc, Ys) :-
    f(X, Y),
    collect_loop(Xs, [Y|Acc], Ys).
Required example 2: fold-compatible nondeterminism
Input:
total(Xs, Total) :-
    findall(V, (member(X, Xs), score(X, V)), Vs),
    sum_list(Vs, Total).
Output:
total(Xs, Total) :-
    total_loop(Xs, 0, Total).
total_loop([], Acc, Acc).
total_loop([X|Xs], Acc0, Total) :-
    score(X, V),
    Acc1 is Acc0 + V,
    total_loop(Xs, Acc1, Total).
Required example 3: spliced templates
Input:
report(Xs, Pairs, Triples) :-
    findall([X,A], (member(X,Xs), expensive(X,A)), Pairs),
    findall([X,A,B], (member(X,Xs), expensive(X,A), other(A,B)), Triples).
Output:
report(Xs, Pairs, Triples) :-
    report_loop(Xs, [], RPairs, [], RTriples),
    reverse(RPairs, Pairs),
    reverse(RTriples, Triples).
report_loop([], Pairs, Pairs, Triples, Triples).
report_loop([X|Xs], Pairs0, Pairs, Triples0, Triples) :-
    expensive(X,A),
    Pairs1 = [[X,A]|Pairs0],
    other(A,B),
    Triples1 = [[X,A,B]|Triples0],
    report_loop(Xs, Pairs1, Pairs, Triples1, Triples).
Required example 4: nested dependent expensive subpredicate
Input:
nested(Xs, Out) :-
    findall(Z,
        ( member(X, Xs),
          expensive(X, A),
          member(Y, A),
          transform(X, Y, Z)
        ),
        Out).
Output:
nested(Xs, Out) :-
    nested_loop(Xs, [], Rev),
    reverse(Rev, Out).
nested_loop([], Acc, Acc).
nested_loop([X|Xs], Acc0, Out) :-
    expensive(X, A),
    nested_inner_loop(X, A, Acc0, Acc1),
    nested_loop(Xs, Acc1, Out).
nested_inner_loop(_, [], Acc, Acc).
nested_inner_loop(X, [Y|Ys], Acc0, Acc) :-
    transform(X, Y, Z),
    nested_inner_loop(X, Ys, [Z|Acc0], Acc).

report(Xs, Pairs, Triples) :-
    findall([X,A], (member(X,Xs), expensive(X,A)), Pairs),

Add safety note:
Plop does not remove nondeterminism blindly. It classifies the meaning of the nondeterminism first. If the code contains cuts, IO, meta-calls, random predicates, var-sensitive predicates, or unknown expensive predicates, Plop either skips the transformation or requires explicit declarations.
